# ChatGPT Style Frontend Redesign Design

## 背景

AIBidder 现有前端虽然已经完成工作台模块拆分，但整体体验仍然偏“本地联调控制台”：首页、导航、模块页、设置抽屉、错误页、加载页和 Copilot 的视觉语言不统一，按钮尺寸与状态样式也存在分散定义。对于非 IT 背景的建筑施工单位招投标工程师来说，当前界面在认知上仍然偏系统后台，而不是“围绕投标任务逐步推进的业务助手”。

本轮重构以 `chatgpt.com` 为模板参考，目标不是复制聊天产品，而是吸收其“安静、聚焦、强层级、低装饰”的界面方法，并结合招投标业务流程，形成更容易上手的投标工作台。

## 目标

1. 将首页、工作台、设置、错误页、加载页统一到一套 minimalist、现代简约、色盲/色弱友好的视觉体系。
2. 将首页改造成“工作台入口型首页”，让用户先选择要推进的投标任务，而不是面对大量杂乱面板。
3. 将页面文案改写为符合非 IT 招投标工程师认知习惯的业务语言，减少技术术语。
4. 将 Copilot 改造成默认隐藏的右侧抽屉，参考 `ExpertDatebase` 的呈现方式，但更贴合 AIBidder 的投标业务场景。
5. 统一按钮、输入框、卡片、状态标签、提示语、布局留白和交互反馈，消除样式漂移。

## 非目标

1. 本轮不重写后端 API 与业务流程。
2. 本轮不将 Copilot 扩展为新的智能代理系统，重点是呈现与交互。
3. 本轮不做大规模状态管理重构，只在必要处收口页面壳层与组件边界。
4. 本轮不强制引入重型 UI 框架；优先使用现有 Next.js + CSS 体系完成统一设计语言。

## 用户画像与文案原则

目标用户为建筑施工单位中的招投标工程师、商务经理、项目投标负责人，通常熟悉资料整理、招标文件分析、资格审查、技术标与商务标编制，但不一定具备 IT 或产品系统使用背景。

因此文案遵循以下原则：

- 用“业务动作词”替代“技术名词”。
- 用“下一步建议”替代“系统状态堆砌”。
- 说明重点放在“做什么、为什么、缺什么”，而不是“接口、模型、运行时配置”。
- 避免英文缩写与后台术语；必要时用工程投标场景化表达。

示例：

- “工作流配置” → “处理步骤设置”
- “Runtime Settings” → “模型与服务设置”
- “Document type” → “资料类别”
- “Connectivity check” → “检查服务是否可用”

## 整体信息架构

### 1. 左侧导航

左侧导航采用 ChatGPT 风格的极简竖向导航，但入口命名全部面向业务动作：

- 首页
- 资料准备
- 招标分析
- 内容编写
- 校核定稿
- 项目归档

底部保留两个固定入口：

- 打开助手
- 设置

导航不堆叠二级菜单，不展示复杂统计数字，不做传统后台树状结构。目标是让用户一眼理解“我接下来要进入哪一步”。

### 2. 首页结构

首页采用“工作台入口型”布局，是本轮重构的核心：

- 顶部轻量标题栏
- 当前项目条
- 中央主引导语
- 任务入口卡片区
- 继续处理区

首页的角色不是展示所有数据，而是帮助用户迅速决定“今天先处理哪一步”。

### 3. 顶部项目条

在主区顶部增加一条高信息密度但低视觉负担的项目条，固定展示：

- 当前项目
- 截止时间
- 当前阶段
- 系统提醒

该区域要像“工作提示条”，不是运营数据看板。用户要一眼知道自己正在处理哪个投标项目、卡在哪一步、还缺什么。

### 4. 模块页

进入具体模块后，页面切换为单任务工作模式，不再展示首页入口卡片。模块页共享统一壳层：

- 顶部标题与说明
- 当前项目条
- 模块专属主操作区
- 次级信息区
- Copilot 抽屉触发入口

每个模块保持独立任务聚焦，避免一个页面堆满多类信息。

## 首页布局设计

首页参考 ChatGPT 的中轴留白布局，但服务于投标场景：

### 主引导区

居中展示一条主引导语，例如：

- 今天先处理哪一步？

辅助说明采用业务口吻：

- 你可以先整理资料，也可以继续上次未完成的投标工作。

### 任务入口卡片

首页展示 6 张统一风格任务卡片：

- 整理投标资料
- 分析招标文件
- 生成投标内容
- 检查评分与废标风险
- 排版并导出
- 归档与复用

每张卡片只保留：

- 标题
- 一句业务说明
- 一个统一主按钮：进入此步骤

不在卡片中堆砌统计图、状态数字和次级按钮，保证认知简洁。

### 继续处理区

在任务卡片下方增加一张“继续处理”横向卡片，提示用户最近一次未完成事项。例如：

- 你上次停留在“招标文件分析”，已提取 18 条关键要求。
- 按钮：继续处理

这块内容用于承接频繁中断、跨天工作的真实投标场景。

## 视觉设计规范

### 1. 视觉气质

整体气质为：

- 现代
- 克制
- 专业
- 可信

避免高饱和色、大面积渐变、厚重阴影和嘈杂装饰，优先使用留白、轻边框、清晰层级来建立秩序。

### 2. 色彩系统

为满足色盲/色弱友好要求，状态信息不依赖红绿对抗，采用“颜色 + 图标 + 文本”三重表达。

推荐语义色：

- 页面背景：暖白 / 低饱和浅灰
- 面板背景：纯白
- 主文字：深灰近黑
- 次级文字：中灰
- 主操作色：低饱和蓝色
- 警示色：琥珀/橙色
- 成功态：蓝绿偏青或中性深色勾选，不直接靠鲜绿色

### 3. 按钮系统

全站统一仅保留三种按钮语义：

- 主按钮：开始、保存、确认、进入此步骤
- 次按钮：查看详情、返回、重新选择
- 文字按钮：展开更多、关闭、删除历史

统一规则包括：

- 高度统一
- 圆角统一
- 字号与字重统一
- 图标尺寸统一
- hover / active / disabled / loading 规则统一

禁止各页面私自定义不同尺寸和不同色相的主按钮。

### 4. 输入与表单

输入框采用大圆角、轻边框、柔和背景。标签语言必须业务化，例如：

- 请填写本次投标项目名称
- 请上传招标文件或相关资料
- 建议先补充资质、人员和业绩材料

错误与提示文案避免技术式报错，优先告诉用户“问题是什么、下一步怎么做”。

### 5. 卡片与容器

卡片样式统一：

- 边框统一
- 圆角统一
- 阴影统一
- 内边距统一
- hover 行为统一

不同区域通过留白和标题区分，而不是靠大量分割线或高对比色块。

## Copilot 设计

Copilot 参照 `ExpertDatebase` 的右侧抽屉形态，但在定位上更克制：它是投标助理，不是持续占据页面的主角色。

### 1. 结构

Copilot 采用右侧抽屉：

- 默认收起
- 点击右下角触发器或左侧导航底部入口后展开
- 窄屏下点击外部区域自动收起

### 2. 默认内容

展开后默认展示三类信息：

- 我现在在处理什么
- 建议下一步
- 常用提问

例如在“招标分析”模块中，常用提问可为：

- 帮我总结资格条件
- 哪些内容最容易废标
- 我下一步要补什么资料

### 3. 对话风格

Copilot 的语言必须像懂招投标业务的助理，而不是技术支持人员。重点是：

- 解释当前步骤
- 提醒缺失资料
- 提示风险点
- 建议下一步

### 4. 功能边界

本轮重点是抽屉交互、文案、上下文感知展示、快捷问题与统一样式，不重写 Copilot 的后端能力和复杂智能编排。

## 页面与组件边界

建议保留现有 Next.js App Router，不新增复杂框架，但对前端结构做如下重整：

### 全局壳层

新增统一壳层组件，负责：

- 左侧导航
- 顶部轻量栏
- 当前项目条
- 右侧 Copilot 触发器与抽屉
- 全局容器宽度与留白体系

### 首页层

首页只负责：

- 任务入口
- 项目提示
- 最近继续处理

### 模块页层

模块页只负责各自业务内容，在统一壳层内部渲染。

### 基础组件层

抽离统一组件，例如：

- `AppShell`
- `ProjectContextBar`
- `TaskEntryCard`
- `PrimaryButton`
- `SecondaryButton`
- `StatusBadge`
- `EmptyState`
- `CopilotDrawer`

## 错误页与加载页

`error.tsx` 与 `loading.tsx` 也应统一到同一视觉语言：

- 更少技术味
- 更强指引性
- 明确“发生了什么”和“可以怎么做”

例如：

- 页面暂时没有打开成功，请重试
- 系统正在准备当前投标工作页面，请稍候

## 可访问性

本轮必须保证：

- 重要状态同时使用文字、颜色与图标表达
- 所有按钮与抽屉存在可见 focus 状态
- Copilot 抽屉、导航、模块切换支持键盘操作
- 对比度满足长时间办公使用场景

## 测试与验证

前端本轮没有新增单元测试体系，验证策略如下：

1. `npm run build` 必须通过。
2. 关键页面至少完成路由级与样式级回归：
   - 首页
   - `/workspace/[module]`
   - 设置抽屉
   - Copilot 抽屉
   - 错误页 / 加载页
3. 必须人工检查：
   - 按钮尺寸与风格是否统一
   - 文案是否符合招投标工程师语境
   - Copilot 是否默认隐藏且能正常展开/收起

## 结论

本轮前端重构的核心不是“继续加功能”，而是把 AIBidder 从“联调控制台”重塑为“面向招投标工程师的工作台”。通过 ChatGPT 风格的壳层、工作台入口型首页、默认隐藏的 Copilot 抽屉、统一的按钮与表单体系，以及更贴合业务的语言表达，系统将更容易理解、更容易上手，也更适合在真实投标工作场景中持续使用。
