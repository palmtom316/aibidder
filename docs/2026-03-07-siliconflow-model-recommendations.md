# 基于硅基流动 (SiliconFlow) 的 AIBidder 角色模型商业建议 (2026最新版)

> **修订说明**：此版本基于 2026 年 3 月硅基流动平台最新的在线模型列表进行全面更新，将原本基于陈旧搜索数据的推荐更正为当前实际可用的最新模型架构（如 Qwen3.5、PaddleOCR-VL-1.5 等）。

根据对 AIBidder 项目架构的分析（特别是 `2026-03-06-phase1-implementation-plan.md` 中定义的 BYOK 默认角色模型矩阵），系统在长文档拆解、受控写作和整稿复核阶段，将 AI 职责细分为了 6 个核心角色。

结合硅基流动 (api.siliconflow.cn) 平台当前最新的模型矩阵与定价，为您提供以下各角色的选型建议：

## 1. OCR 与视觉读取角色 (`ocr_role`)
* **职责定位**：负责跨页表格、评分表、资格表等高难度 PDF 页面的高保真还原。
* **硅基流动最新建议**：
  * **主选推荐（极高性价比）**：`PaddlePaddle/PaddleOCR-VL-1.5`（**免费**）
  * **备选/复杂视觉推荐**：`Qwen3-VL` 系列 或 `deepseek-ai/DeepSeek-OCR`（**免费**）
  * **推荐理由**：硅基流动上线了百度最新的 `PaddleOCR-VL-1.5` 和 `DeepSeek-OCR`（均为免费提供），对文档的多模态理解和表格、印章识别非常强悍，完全能够取代旧版的通用视觉大模型，无缝集成到第一步的高保真解析中。

## 2. 拆解导航角色 (`decomposition_navigator_role`)
* **职责定位**：负责分析招标文件全文、进行章节导航、前置任务的分派以及复杂跨章节的逻辑判断。需要极强的上下文理解（长文本处理）和逻辑决策能力。
* **硅基流动最新建议**：
  * **主选推荐**：`deepseek-ai/DeepSeek-V3` 或 `Pro/deepseek-ai/DeepSeek-V3` 
  * **备选推荐（长文本专精）**：`moonshotai/Kimi-K2-Instruct-0905` 或 `Pro/MiniMaxAI/MiniMax-M2.1`
  * **推荐理由**：DeepSeek-V3 带有“长文本处理”和“旗舰全能”标签，是梳理整份招标书结构的完美选择。若是企业级商用，建议切入其官方提供的 `Pro/deepseek-ai/DeepSeek-V3` 节点以获得更稳定的高并发支持。Kimi-K2 则在极长上下文检索中表现优异。

## 3. 并行抽取角色 (`decomposition_extractor_role`)
* **职责定位**：负责对招标文件中的基础信息、资格要求、评分标准等 7 类要素进行高并发的细粒度提取。任务种类多、调用频率极高，对 JSON 格式遵循度和单次调用成本极其敏感。
* **硅基流动最新建议**：
  * **极限控本/平替推荐**：`Qwen/Qwen3.5-4B`（**免费**）
  * **高性能抽取推荐**：`Qwen/Qwen3.5-9B`（极低成本：输入￥1.5/M，输出￥12/M）或 `GLM-4.5-Air`
  * **推荐理由**：旧版 Qwen 2 系列已全面被 2026年3月最新发布的 **Qwen 3.5 系列**取代。其中 Qwen3.5-4B 在硅基流动完全免费，对于局部切片文本的信息抽取如“抽取工期、提取过期标记”绰绰有余；如果格式要求高，可上 Qwen3.5-9B，极速且价格便宜。

## 4. 标书写作角色 (`writer_role`)
* **职责定位**：基于检索到的证据包、企业事实库进行受控表达和段落重写。要求绝不胡编乱造（幻觉极低），且文风严谨。
* **硅基流动最新建议**：
  * **主选推荐**：`deepseek-ai/DeepSeek-V3`
  * **备选推荐**：`moonshotai/Kimi-K2-Instruct-0905`（带有“文案创作”最佳标签）
  * **推荐理由**：受控写作绝不能使用普通的轻量模型。DeepSeek-V3 稳定且智能；而 Kimi-K2-Instruct 则是在中文正规商务文案和公文写作上有得天独厚的优势。

## 5. 合规与一致性审核角色 (`reviewer_role`)
* **职责定位**：对生成的招标文件进行重型 Agent 维度的串联审查，包括合规性、事实一致性检查、废标项核对等。需要非常深度的数学推理和逻辑思维链能力。
* **硅基流动最新建议**：
  * **主选推荐**：`deepseek-ai/DeepSeek-R1` 或 `Pro/deepseek-ai/DeepSeek-R1`
  * **推荐理由**：硅基流动的 R1 模型打上了“数学推理”与“长文本处理”标签。对招标合同中的付款节点、违约金比例计算、金额不一致审查，以及漏项、废标项核对，R1 的强化学习思维链是绝对权威。

## 6. 最终裁决总结角色 (`adjudicator_role`)
* **职责定位**：接收多个 Reviewer Agents 输出的问题单，进行汇总去重，输出结构化定稿建议。
* **硅基流动最新建议**：
  * **主选推荐**：`deepseek-ai/DeepSeek-V3` 或 `Pro/MiniMaxAI/MiniMax-M2.1`
  * **推荐理由**：这一步主要是大范围信息综合。带有 "Vibe Coding" 以及极佳结构化处理能力的 DeepSeek-V3 或 MiniMax-M2.1 能最快速地完成复杂的多 JSON 问题单合并与裁决。对于系统最终生成 API 响应十分平滑。

---

## 策略落地总结

在依托硅基流动构建本平台时，结合 2026 年 3 月最新模型矩阵的**分层调度 (Tiered Routing)** 策略：
1. **白嫖高并发层（Extractor/OCR 等脏活累活）**：充分利用硅基平台提供的免费端点，如 `Qwen3.5-4B`（文本抽取）和 `PaddleOCR-VL-1.5`（多模态解析），极大压缩日常运行流水线成本。
2. **高端心智层（Navigator/Reviewer）**：在核心的章节调度、逻辑定性与最终复核环节，使用 `DeepSeek-R1/V3` 与 `Kimi-K2`，保障标书最核心的质量与风险守门。
3. **企业级稳定选项**：对于高 SLA 要求的正式商用上线，建议对 R1/V3 切入 `Pro` 级别节点（如 `Pro/deepseek-ai/DeepSeek-V3`）以保证并发稳定不掉线。
